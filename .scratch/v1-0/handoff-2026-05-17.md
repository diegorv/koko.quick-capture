## Known good-to-know patterns

- **Windows are hidden-then-shown, not destroyed-and-recreated.** All three windows (Inbox, Composer, Dock) intercept `WindowEvent::CloseRequested { api, .. }` and call `api.prevent_close() + window.hide()`. Don't drop this; it was a bug class.
- **Composer focus on each reopen**: the route listens for `open_composer` events from Rust and bumps a `focusKey` prop that the Composer component watches with `$effect` to clear text and refocus the textarea. Necessary because the component only mounts once.
- **macOS `show + set_focus` must run on the main thread** via `app.run_on_main_thread(...)`. The global-shortcut plugin and tray handlers run on workers. Every show/focus path in `lib.rs` already hops.
- **Event names use `:` as the hierarchical separator** (see `9097c3a`). `captures:changed`, `dock:pulse`, `tray:open_inbox`, etc. Tauri 2 rejects dots.
