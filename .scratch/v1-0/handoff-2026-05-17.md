## What landed between v0.3.0 and v0.3.1

- Full-text **search** inside the Inbox: FTS5 virtual table
  (`captures_fts`) populated by `Store::save` / cleared by
  `Store::soft_delete`. `Store::search` runs `MATCH ?` with a
  sanitised query (`build_fts_match` splits on non-alphanumeric
  runs, prefix-matches each token). New `search_captures` command.
- Inbox UI gains a live search bar above the list pane; 150ms
  debounce; `Cmd+F` focuses + selects; `Esc` clears. Search bar is
  a sibling of `.list-pane`, not inside it — keeps the scroll
  container free of overlays so Tab-focus into the listbox can
  never tuck rows underneath. Status bar swaps "N captures" for
  "K results" while searching. Pagination disabled while searching.
- `commands::is_clipboard_duplicate` — clipboard shortcut now skips
  the save + pulse + emit when the latest stored capture has the
  same payload (Clip / Note text, Link url, File source_path, Shot
  path; binary Shot { Bytes } is never deduped).
- Dock right-click context menu renders Lucide icons (same glyphs
  as the Tray menu) without accelerator hints.
- Dock pulse `onanimationend` resets `pulsing` so consecutive saves
  always animate.
- Arrow-key nav `scrollIntoView({ block: "nearest" })` keeps the
  selected row in view past the visible window.
- Menu dispatch unified behind `dispatch_menu_item` (Tray + Dock
  right-click share the same arms).
- `PREV_FRONTMOST_PID` moved to `app.manage(PrevFrontmostPid)`.
- Schema migration drops the v0.2 `read_at` backfill — fresh DBs
  only. Existing users must `rm -rf
  ~/Library/Application\ Support/com.koko.quick-capture` before
  relaunching v0.3.1.
- All v1.0-scope polish items closed; the `.icns` shipped at v0.2;
  v0.3 adds clipboard dedup + dock right-click icons + a central
  event-name registry, plus a clutch of post-release bug fixes
  (Cmd+Tab focus return, double-save guard, animation restart).
- The repo is local-only (`git remote -v` empty). Nothing pushed.

## What landed between v0.2.0 and v0.3.0

- `commands::dismiss_composer` returns focus to the prior
  macOS app via NSRunningApplication.activate (tracks PID via
  NSWorkspace before each Composer summon). Earlier `[NSApp hide:]`
  attempt was reverted because it auto-restored every previously
  visible window on the next `window.show()`.
- `commands::show_composer` / `commands::show_inbox` /
  `intercept_close_as_hide` extracted so every entry point (shortcut,
  tray, dock context menu, JS invoke) routes through the same
  helper.
- `commands::capture_clipboard_and_broadcast` + free fn
  `dispatch_shortcut(app, id)` shrink the global-shortcut handler
  closure to a one-liner.
- `is_clipboard_duplicate` suppresses re-saving when the user hits
  the clipboard shortcut twice with the same payload (Clip / Note /
  Link / File / Shot { path }; Shot { Bytes } is intentionally not
  deduped).
- `events.rs` + `src/lib/events.ts` hold every Tauri bus-event name;
  every emit / listen site imports a constant.
- `Dock.svelte` uses `onanimationend` to reset `pulsing` so
  consecutive saves animate.
- Composer save guards against re-entry on rapid Cmd+Enter.
- Inbox `Esc / Cmd+W` routes through a new `hide_inbox` Rust command
  that mirrors the CloseRequested handler (window.hide + revert
  activation policy to Accessory).
- Dock right-click menu renders Lucide icons (no accelerator hints)
  to match the Tray menu visually.

  (size + position flags only; `VISIBLE` is excluded so hidden
  windows do not pop on relaunch).

## Patterns that still apply

- **Windows are hidden-then-shown, never destroyed.** All three
  windows intercept `WindowEvent::CloseRequested { api, .. }` and
  call `api.prevent_close() + window.hide()`.
- **Composer refocus on reopen**: parent route bumps a `focusKey`
  prop the Composer watches via `$effect`. Necessary because the
  component only mounts once.
- **macOS `show + set_focus` must run on the main thread** via
  `app.run_on_main_thread(...)`. Same applies to
  `set_activation_policy`.
- **Event names use `:` as the hierarchical separator.** Tauri 2's
  `IllegalEventName` regex rejects `.`. Active names:
  `captures:changed`, `dock:pulse`, `dock:unread:changed`,
  `dock:drag:enter`, `dock:drag:leave`, `dock:fullscreen:entered`,
  `dock:fullscreen:exited`, `tray:open_inbox`, `open_composer`,
  `open_inbox`.
- **Listbox autofocus**: `InboxList.svelte` grabs window focus on
  cold open and re-grabs on row click + window-focus so arrow keys
  work without an extra Tab.
