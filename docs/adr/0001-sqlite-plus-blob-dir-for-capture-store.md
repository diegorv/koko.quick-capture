# SQLite plus blob directory for the Capture store

The app needs to persist heterogeneous Captures (Link, Clip, Shot, File, Note) for fast listing, filtering, and future programmatic processing. We picked a hybrid local store: a SQLite database holds one row per Capture (id, kind, created_at, source metadata, JSON payload), and a sibling `blobs/` directory holds the original files for `Shot` and `File` Captures, referenced from the DB by path. Markdown-vault and pure-JSON alternatives were considered and rejected: a markdown vault would make Obsidian-style inspection easy but loses transactional writes and structured queries that the inbox UI and any future automation will need, and a JSON log file does not scale to mutations. A one-way export to markdown can be added later without changing this store.
