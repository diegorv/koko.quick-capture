# Composer hosts a CodeMirror editor instead of a plain textarea

The Composer previously rendered a single `<textarea>` because its only job was to accept a Note Capture's free text. Adding `[[` autocomplete against a configurable [Wikilink source folder](../../CONTEXT.md#wikilink-source-folder) requires a popup anchored to the caret with keyboard navigation (↑↓ to move, Enter to pick, ESC to dismiss, live filtering as the user types), and that surface does not exist on `<textarea>` without hand-rolling caret-coordinate math, key precedence, and popup positioning. Swapping the Composer's editor host to a minimal CodeMirror 6 setup (`EditorView` + `EditorState` + `autocompletion` + `keymap` + `placeholder`, no markdown highlighting, no live-preview) buys all of that for free via `@codemirror/autocomplete`, including the precedence with Cmd+Enter (still saves) and ESC (closes popup first, then the window). The completion source is a near-port of brain's `wikilinkCompletion` — same `detectWikilinkContext` logic for "is the cursor inside an open `[[`", same insert behavior of `[[Name]]` with auto-closing `]]` and cursor-after — so the two apps stay in sync on wikilink mechanics.

A plain textarea with a custom popup was considered and rejected: textarea-caret-position tricks are fragile under IME and resize, key precedence has to be reimplemented by hand, and the resulting code is larger than the CM setup it replaces. BlockNote was considered and rejected: it is a block-based WYSIWYG editor on top of ProseMirror, which forces a block model + markdown serialization layer for what is, in the Capture domain, plain `text`; the bundle is several times larger and the "single text field, ESC cancels, Cmd+Enter saves" Composer contract does not survive a block model intact. A full port of brain's markdown-editor (with markdown language, live-preview, decorations, embeds) was rejected because the Composer is not a writing studio — per CONTEXT.md it is a quick-entry surface whose single job is to accept a Note Capture and disappear, and richer editing belongs elsewhere if it belongs in QC at all.

The cost of this ADR is the CodeMirror dependency, a one-time rewrite of `Composer.svelte` and `Composer.test.ts`, and the Tauri wiring for the new Rust commands that back autocomplete and settings (`list_people`, `pick_folder`, settings get/set exposed as `#[tauri::command]`). The reward is a Composer that produces wikilink-bearing Captures with a polished autocomplete, sharing its completion logic with brain so future cross-app integration is a flat handoff rather than a parsing exercise.
