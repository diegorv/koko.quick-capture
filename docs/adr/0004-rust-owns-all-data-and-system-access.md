# Rust owns all data and system access

The Svelte frontend has no direct access to SQLite, the clipboard, the filesystem, or global shortcuts. Every data or OS interaction goes through a typed Rust command invoked via `tauri::command` / `invoke()`. The Rust side uses `rusqlite` directly (not `tauri-plugin-sql`), the `ulid` crate for IDs, and Tauri's `global-shortcut` and `clipboard-manager` plugins (or `arboard`) for system surfaces. `tauri-plugin-sql` was rejected because letting JS write SQL strings tightly couples the UI to the schema and complicates future migrations; routing everything through Rust commands gives a stable, typed IPC surface that the UI consumes without knowing SQL exists, and keeps capture-write logic (ULID generation, blob persistence, URL detection) in one place.
