# Full test coverage for Rust and Svelte code

Every Rust module and every Svelte component shipped in this project ships with tests in the same slice. There is no "v0.1 manual-smoke exception" for UI or thin wiring code. Rust tests use the stdlib `#[test]` harness with integration tests under `src-tauri/tests/`; Svelte components are tested with Vitest plus `@testing-library/svelte`. End-to-end browser tests (Playwright or equivalent) are deferred until a clear need arises — the policy here is unit and component coverage, not e2e. Selective coverage was rejected because v0.1 is a tracer bullet whose whole job is to validate the integration shape, and untested wiring would defeat the verification we built it for; Playwright was rejected for now because its setup cost outweighs its v0.1 value while no real UI flow yet justifies an e2e harness. Modules that are pure thin wrappers around platform APIs (e.g. global shortcut registration) test the seam — extract a testable intent registry or dispatcher and test that — rather than asserting against the real OS hook, which is fine to verify by manual smoke.
